# MatchIntel Community System 0.7.4

This release updates the backend, Discord bot, and website. The MatchIntel client remains version 0.7.1 and does not need a code change for this release.

## What changed

- Every timed key now keeps its full duration until its first successful activation in the MatchIntel app.
- Timed keys generated by the Discord bot, website free-trial flow, or backend CLI all use activation-based timing.
- Existing timed keys that have no device or refresh-token evidence are migrated to pending activation automatically.
- Existing keys that have already been activated keep their current expiration time.
- The Discord bot adds the DM-only `/whatsmytrialkey` command.
- `/whatsmytrialkey` returns the exact encrypted key originally issued by the website. It never generates a second trial.
- `/whatsmykey`, staff key views, and generated-key messages now clearly show `Awaiting first activation` instead of incorrectly showing `Lifetime` for an unused timed key.
- The free-trial result page explains that the timer starts on first activation and tells users how to recover the same key later.
- The Discord ticket types from bot 0.7.3 remain included.

## Deploy order

1. Deploy the `backend` folder first. Railway will automatically run `008_activation_timed_licenses.sql`.
2. Wait until `/health` and `/ready` report backend version `0.7.4` and the database is ready.
3. Deploy the `discord-bot` folder. The global `/whatsmytrialkey` command may take a short time to appear everywhere in Discord.
4. Deploy the `website` folder so the trial result page matches the new activation timing.

No new environment variables are required. Keep `JWT_SECRET`, `LICENSE_KEY_ENCRYPTION_KEY`, `ADMIN_API_KEY`, `WEBSITE_API_KEY`, and every existing database variable unchanged.

## Important migration behavior

Migration 008 only resets an existing timed key to pending activation when there is no device record and no refresh-token record for that license. This prevents previously used keys from accidentally receiving a fresh timer after device resets or support actions.

## Verification checklist

1. Generate a trial key with `/genkey` and confirm it says `Awaiting first activation`.
2. Wait a few minutes and confirm the key still has no expiration timestamp.
3. Activate it in the app and confirm the full duration begins at that moment.
4. Claim a website trial, close the page, DM the bot, and run `/whatsmytrialkey`.
5. Confirm the bot returns the exact same key shown on the website.
6. Run `/whatsmytrialkey` in a server and confirm the bot tells the user to use DMs.
7. Confirm the user cannot claim a second website trial.
